% The field of Artificial Intelligence in Education creates intelligent tutoring systems that push the technological boundary of e-learning. The field distinguishes itself from others for its hard focus on using Artificial Intelligence to imbue regular software with smart and adaptive behaviours. Sharing computational representation of students known as student models, has long been a problem of interest for this community. The interest comes from the fact that sharing these models reduces the cost and time invested in the development of new tools. Current systems for sharing student models can share traditional student models, but can't share student models that have machine learning components. In this thesis I propose two ways of solving this problem. The first is a multi-agent architecture for decoupled student models that enables machine learning components to: a) train each model with independent data, b) choose what data is more reliable for each student model, and c) organize and learn through time which is the more appropriate model to work with each tutor. The architecture uses a fragmented user model approach and it can share student models that have machine learning components. The second solution is set of metrics that help researchers share models without systems or formal mechanisms, i.e., manually. This second approach is based on an analysis of the portability of current state of the art student models. Both the architecture and the portability analysis present smaller contributions such as the protocols and self-organizing mechanisms used in the architecture, and the student model extraction and implementation guidelines for portability from the analysis. At the end of this book I present a discussion of the importance of sharing intelligent components between programs and elaborate in why this work is a stepping stone towards smarter better ways of developing intelligent tools that imbue traditional software with intelligence.\\
